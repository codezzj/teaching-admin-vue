## 一、项目概述

### 1.1 项目背景

本课程设计为《Web前端框架技术》课程的综合性实战项目，要求综合运用课程所学的 Vue3、Vite、前端工程化、组件、路由、UI 框架等核心技术，独立完成"智能教学辅助管理系统"的前端开发。

### 1.2 设计目标

1. 熟练掌握 Vite 搭建 Vue3 项目的工程化流程
2. 精通 Vue3 组合式 API 开发方式
3. 掌握 Vue Router 5 路由配置、路由守卫、权限控制
4. 熟练使用 Element Plus 组件库实现页面布局与交互
5. 掌握 Axios 网络请求封装、Mock 数据模拟、表单验证、分页/搜索等通用功能
6. 具备组件化封装、代码规范编写、项目文档撰写的基础能力
7. 实现一个功能完整、交互友好、可直接运行的教学管理类 Web 系统

### 1.3 系统定位

本系统定位为**教学辅助管理平台（教师端）**，为教师提供学生信息管理、课程管理、作业发布、考勤签到、题库维护、测试批改等一站式教学管理功能。前端所有数据均通过 vite-plugin-mock 模拟后端接口获取，系统可独立运行、无需后端服务。

### 1.4 开发环境

- 操作系统：Windows
- 开发工具：VS Code
- Node.js 版本：18+
- 包管理器：npm

## 四、关键技术实现

### 4.1 Axios 请求封装

本项目中所有网络请求均通过统一的 `request.js` 模块发出，核心封装思路如下：

```javascript
// src/utils/request.js
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { getToken, clearAuth } from './auth'

const request = axios.create({
    baseURL: '/api',
    timeout: 10000
})

request.interceptors.request.use((config) => {
    const token = getToken()
    if (token) config.headers.Authorization = `Bearer ${token}`
    return config
})

request.interceptors.response.use(
    (resp) => {
        const res = resp.data
        if (res && typeof res.code !== 'undefined') {
            if (res.code === 0) return res.data
            ElMessage.error(res.message || '请求失败')
            return Promise.reject(res)
        }
        return res
    },
    (err) => {
        if (err?.response?.status === 401) {
            clearAuth()
            window.location.href = '/#/login'
        } else {
            ElMessage.error(err?.message || '网络错误')
        }
        return Promise.reject(err)
    }
)

export default request
```

**关键设计点：**
- **请求拦截器**：自动从 localStorage 读取 Token 并注入到请求头 `Authorization` 字段中
- **响应拦截器**：统一处理接口返回格式 `{ code, message, data }`，code=0 视为成功，否则弹出错误提示；401 状态自动清除登录态并重定向登录页
- **统一 baseURL**：所有接口基础路径设置为 `/api`，与 vite-plugin-mock 的拦截路径保持一致
- **超时控制**：设置 10 秒超时，防止请求挂起

### 4.2 路由守卫实现

本项目的路由守卫集中配置在 `src/router/index.js` 中：

```javascript
// src/router/index.js
import { createRouter, createWebHashHistory } from 'vue-router'
import { getToken } from '../utils/auth'

const routes = [
  {
    path: '/login',
    component: Login,
    meta: { title: '登录', requiresAuth: false }
  },
  {
    path: '/',
    component: AdminLayout,
    meta: { requiresAuth: true },
    children: [
      { path: '', redirect: '/dashboard' },
      { path: 'dashboard', component: Dashboard, meta: { title: '首页' } },
      { path: 'students', component: StudentsList, meta: { title: '学生管理' } },
      { path: 'courses', component: CoursesList, meta: { title: '课程管理' } },
      // ... 其他路由
    ]
  }
]

const router = createRouter({
  history: createWebHashHistory('/teaching-admin-vue'),
  routes
})

// 全局前置守卫
router.beforeEach((to) => {
  const token = getToken()
  if (to.path === '/login') {
    return token ? '/dashboard' : true
  }
  if (to.meta.requiresAuth && !token) {
    return '/login'
  }
  return true
})

export default router
```

**关键设计点：**
- 使用了 `beforeEach` 全局前置守卫
- 业务页面路由通过 `meta: { requiresAuth: true }` 标记需要登录
- 守卫逻辑：访问需要登录的页面但无 token → 重定向 `/login`；已登录访问 `/login` → 重定向 `/dashboard`

### 4.3 vite-plugin-mock 数据模拟

本项目使用 vite-plugin-mock 模拟后端接口，所有业务页面的数据均通过 axios 请求获取，未使用任何直接写死的假数据。以下以学生管理接口为例：

```javascript
// mock/students.js
function nowString() {
    const d = new Date()
    const pad = (n) => String(n).padStart(2, '0')
    return `${d.getFullYear()}-${pad(d.getMonth() + 1)}-${pad(d.getDate())} ${pad(d.getHours())}:${pad(d.getMinutes())}`
}

let students = Array.from({ length: 27 }).map((_, i) => {
    const idx = i + 1
    return {
        id: idx,
        studentNo: `2025${String(idx).padStart(4, '0')}`,
        name: `学生${idx}`,
        className: `计科${(idx % 3) + 1}班`,
        createdAt: nowString()
    }
})

export default [
    // 列表：分页 + keyword 模糊搜索
    {
        url: '/api/students',
        method: 'get',
        response: ({ query }) => {
            const page = parseInt(query?.page || '1', 10)
            const pageSize = parseInt(query?.pageSize || '10', 10)
            const keyword = String(query?.keyword || '').trim()

            let filtered = students
            if (keyword) {
                filtered = students.filter((s) =>
                    [s.studentNo, s.name, s.className].some((v) => String(v).includes(keyword))
                )
            }

            const total = filtered.length
            const start = (page - 1) * pageSize
            const list = filtered.slice(start, start + pageSize)

            return { code: 0, message: 'ok', data: { list, total } }
        }
    },

    // 新增
    {
        url: '/api/students',
        method: 'post',
        response: ({ body }) => {
            const { studentNo, name, className } = body || {}
            if (!studentNo || !name || !className) {
                return { code: 1, message: '学号/姓名/班级不能为空', data: null }
            }
            if (students.some((s) => s.studentNo === studentNo)) {
                return { code: 1, message: '学号已存在', data: null }
            }

            const id = students.length ? Math.max(...students.map((s) => s.id)) + 1 : 1
            const item = { id, studentNo, name, className, createdAt: nowString() }
            students.unshift(item)
            return { code: 0, message: 'ok', data: item }
        }
    },

    // 编辑：/api/students/:id
    {
        url: /\/api\/students\/\d+$/,
        method: 'put',
        response: ({ url, body }) => {
            const id = parseInt(url.split('/').pop(), 10)
            const idx = students.findIndex((s) => s.id === id)
            if (idx === -1) return { code: 1, message: '学生不存在', data: null }

            const { studentNo, name, className } = body || {}
            if (!studentNo || !name || !className) {
                return { code: 1, message: '学号/姓名/班级不能为空', data: null }
            }

            // 学号唯一校验（排除自己）
            if (students.some((s) => s.studentNo === studentNo && s.id !== id)) {
                return { code: 1, message: '学号已存在', data: null }
            }

            students[idx] = { ...students[idx], studentNo, name, className }
            return { code: 0, message: 'ok', data: students[idx] }
        }
    },

    // 删除：/api/students/:id
    {
        url: /\/api\/students\/\d+$/,
        method: 'delete',
        response: ({ url }) => {
            const id = parseInt(url.split('/').pop(), 10)
            students = students.filter((s) => s.id !== id)
            return { code: 0, message: 'ok', data: true }
        }
    }
]
```

**key 设计思路：**
- 所有接口返回格式统一为 `{ code: 0, message: 'ok', data: ... }`
- 使用 JavaScript 数组在内存中维护数据，实现增删改查
- 列表接口支持 query 参数分页和关键词搜索

### 4.4 组合式 API 典型用法

本项目中所有页面均采用 Vue3 组合式 API（Composition API）编写，使用 `<script setup>` 语法简化代码。以下是学生的增删改查页面中的典型用法：

```javascript
// src/views/students/StudentsList.vue（关键逻辑）
<script setup>
import { onMounted, reactive, ref } from 'vue'
import { ElMessage, ElMessageBox } from 'element-plus'
import { getStudents, createStudent, updateStudent, deleteStudent } from '../../api/students'

// 响应式状态
const loading = ref(false)
const saving = ref(false)
const keyword = ref('')
const page = ref(1)
const pageSize = ref(10)
const total = ref(0)
const list = ref([])

// 弹窗状态
const dialogVisible = ref(false)
const dialogMode = ref('create')
const editingId = ref(null)
const formRef = ref()

// 表单数据（对象类型用 reactive）
const form = reactive({
  studentNo: '',
  name: '',
  className: '',
  createdAt: ''
})

// 表单校验规则
const rules = {
  studentNo: [{ required: true, message: '请输入学号', trigger: 'blur' }],
  name: [{ required: true, message: '请输入姓名', trigger: 'blur' }],
  className: [{ required: true, message: '请输入班级', trigger: 'blur' }]
}

// 异步获取列表
async function fetchList() {
  loading.value = true
  try {
    const res = await getStudents({
      page: page.value,
      pageSize: pageSize.value,
      keyword: keyword.value
    })
    list.value = res.list
    total.value = res.total
  } finally {
    loading.value = false
  }
}

// 保存（新增/编辑）
async function onSave() {
  await formRef.value.validate()
  saving.value = true
  try {
    if (dialogMode.value === 'create') {
      await createStudent({...form})
      ElMessage.success('新增成功')
    } else {
      await updateStudent(editingId.value, {...form})
      ElMessage.success('保存成功')
    }
    dialogVisible.value = false
    fetchList()
  } finally {
    saving.value = false
  }
}

// 生命周期钩子
onMounted(() => {
  fetchList()
})
</script>
```

**组合式 API 优势体现：**
- `ref` 和 `reactive` 区分基本类型和对象类型的响应式数据
- `<script setup>` 语法使代码更简洁，无需 `return` 导出
- 逻辑集中在 `setup` 中，便于理解和复用

### 5.1 收获与成果

通过本次课程设计，我完成了"智能教学辅助管理系统（教师端）"的完整前端开发，系统包含 **9 个功能模块**（首页 + 8 个业务模块），技术栈覆盖 **Vite + Vue3 + Vue Router 5 + Element Plus + Axios + vite-plugin-mock**。

主要收获如下：

1. **掌握了 Vite 工程化开发流程**：从项目初始化、依赖安装、目录规划到运行调试，熟悉了现代前端项目的完整搭建过程。

2. **深入理解 Vue3 组合式 API**：通过编写 10 余个页面组件，熟练运用 `ref`、`reactive`、`computed`、`watch`、`onMounted` 等核心 API，体会到组合式 API 相比 Options API 在逻辑组织和代码复用上的优势。

3. **熟练使用 Element Plus 组件库**：在项目中充分使用了 el-table、el-form、el-dialog、el-card、el-pagination、el-tabs、el-select、el-date-picker 等组件，对组件的事件监听、插槽使用、表单校验有了全面掌握。

6. **提升了代码规范和工程化意识**：按照 `api/`、`components/`、`views/`、`router/`、`utils/` 的分层结构组织代码，遵循了"单向数据流 + 接口驱动渲染"的开发原则。

### 5.2 不足与改进方向

1. **状态管理**：当前项目未使用 Pinia 进行全局状态管理，部分模块之间存在数据重复请求的情况。后续可引入 Pinia 模块化拆分，提升数据共享效率。

2. **响应式适配**：目前布局主要面向桌面端，对移动端和平板的适配尚未完善。后续可使用 Element Plus 的栅格系统或 Tailwind CSS 进行响应式优化。

3. **数据持久化**：由于使用 vite-plugin-mock，所有数据在刷新页面后重置。后续可接入真实后端或使用 localStorage 进行伪持久化。

4. **单元测试**：当前未编写测试用例。后续可引入 Vitest 对工具函数和组件的关键逻辑进行单元测试。

5. **权限管理**：当前系统仅实现了教师端，三角色权限（管理员/教师/学生）的动态菜单和路由隔离未做详细实现。
